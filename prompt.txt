We need to fix a React state management error where a child component (GalleryManager) is attempting to update the state of a parent component (CreateProjectForm) during render. This causes the error “Cannot update a component while rendering a different component” and leads to unstable UI behavior.

First, identify all places inside GalleryManager where setGalleryState or any parent state setter is being called. Ensure that no state updates happen directly inside the render body, including inside JSX, loops, or conditional rendering.

State updates must only occur inside:

event handlers (e.g., onClick, onChange)
useEffect hooks
async callbacks

Remove any direct state updates from the render phase.

Next, move all data initialization logic (such as setting gallery state after fetching media) into a useEffect hook. This ensures that state updates happen after the component has rendered.

Example pattern:

useEffect(() => {
if (!projectId) return;

async function fetchMedia() {
const { data } = await supabase
.from('project_media')
.select('*')
.eq('project_id', projectId);

if (!data) return;

const grouped = transformData(data);

setGalleryState(grouped);

}

fetchMedia();
}, [projectId]);

Do not call setGalleryState directly inside the component body.

Also ensure that child components do not automatically push updates to parent state on every render. Instead, only trigger updates in response to user actions such as selecting a file or deleting an item.

If GalleryManager needs to initialize state, it should either:

manage its own local state and sync upward on submit
OR
receive initial state from parent and only send updates via controlled events

Avoid patterns where child computes data and immediately pushes it to parent during render.
